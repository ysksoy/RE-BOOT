# 就職浪人・既卒特化型 インターン求人検索メディア（仮）開発構想書

## 1. サービスの概要と目的

* **サービスコンセプト:**
* 「就職浪人（空白期間）」を、ネガティブな停滞期間ではなく、「スキルを身につける助走期間（プレ社会人）」と再定義する。
* エージェント（面談）を挟まず、アルバイト情報誌のように誰でも気軽に「既卒・浪人生歓迎」の長期インターンや求人を検索できるWebメディア。


* **ターゲット:**
* 就職先が決まらないまま卒業した既卒生、または留年して就活を継続している学生。


* **解決する課題:**
* 大手ナビサイトでは「新卒」以外が弾かれやすい。
* 既卒特化サービスは「営業・施工管理」などに偏りがあり、エージェント面談が必須でハードルが高い。
* 「社会との接点（所属）」がない孤独感を、インターンという「所属」で解消する。



---

## 2. システムアーキテクチャ（技術構成）

モダンな「Serverless」かつ「Headless」な構成を採用。低コスト・爆速・高メンテナンス性を実現します。

### 全体像

> **[求人データ源]** → **[収集・解析(Python)]** → **[DB(Supabase)]** → **[API]** → **[Webサイト(Next.js/Vercel)]**

### 採用する技術スタックと選定理由

#### A. フロントエンド（ユーザーが見る画面）

* **Framework:** **Next.js (React)**
* **理由:** ページ表示速度が圧倒的に速く（SEOに強い）、ユーザー体験が良い。今のWeb開発のスタンダード。


* **Hosting:** **Vercel**
* **理由:** Next.jsの開発元が提供するサーバー。GitにPushするだけでデプロイ・更新が完了する。個人開発なら無料で独自ドメイン運用が可能。


* **Domain:** **独自ドメイン (.com / .jp)**
* **管理:** お名前.com / Xserverドメイン / Cloudflare などで取得し、DNS設定でVercelに向ける。



#### B. バックエンド・データベース（データの保存）

* **BaaS:** **Supabase**
* **役割:** 収集した求人情報の保存（PostgreSQL）と、それをフロントエンドに渡すためのAPI提供。
* **理由:** サーバー構築不要。管理画面が使いやすく、無料枠が大きい。認証機能（ログイン）が必要になった際も即対応可能。



#### C. データ収集・バッチ処理（裏側の仕組み）

* **Language:** **Python**
* **Library:** `Playwright` (スクレイピング用), `LangChain` + `OpenAI API` (データ整形・判定用)


* **Execution:** **GitHub Actions**
* **役割:** 毎日決まった時間にスクリプトを自動実行する（Cron実行）。
* **理由:** サーバーを常時起動する必要がなく、コストがかからない（月2000分まで無料）。



#### D. 判定ロジック（品質管理）

* **AI Filter:** **LLM (OpenAI API / Gemini API)**
* **役割:** スクレイピングした求人本文をAIに読ませる。
* **処理内容:** 「この求人は就職浪人生（既卒）でも応募可能か？」「スキルが身につく職種か？」を0〜100点でスコアリングし、適合する求人のみDBに保存する。

## 3. 実装・運用ロードマップ

### Phase 1: データ収集と基盤構築（Python & Supabase）

1. **データソースの選定:**
* まずは「ASP（アフィリエイト）」で公式データ（CSV等）が提供されているか確認。
* ない場合、技術検証用のターゲットサイト（Wantedly等）を1つ決める。


2. **スクリプト作成:**
* Pythonで「タイトル・本文・URL」を取得するコードを書く。
* AI APIを通して「就職浪人OKフラグ」を立てる処理を組み込む。


3. **DB保存:**
* Supabaseにデータを流し込む（Insert）。
* すでに掲載終了した求人を削除（論理削除）する処理を作る。



### Phase 2: Webサイト構築（Next.js & Vercel）

1. **デザイン・コーディング:**
* 「職種検索」「エリア検索」などのUIを実装。
* **Headless UI**や**Tailwind CSS**を使って、モダンで見やすいデザインにする。


2. **SEO対策:**
* `Next.js`の機能で、動的にメタタグ（title, description）を生成する。
* `sitemap.xml`の自動生成、構造化データ（Google for Jobs用）の埋め込み。


3. **デプロイ:**
* Vercelにデプロイし、独自ドメインを接続して公開。



### Phase 3: 法務・運用（Legal & Ops）

1. **規約・法律のクリア:**
* 直リンク方式（送客モデル）を採用し、元サイトの権利を侵害しない設計にする。
* `robots.txt`の遵守と、アクセス頻度の制御（Sleep処理）を徹底する。


2. **自動化:**
* GitHub Actionsで「1日1回深夜に更新」のサイクルを回す。
